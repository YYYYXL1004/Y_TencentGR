## 模型训练

### 1. 环境变量

平台提供了训练过程中需要用到的目录（例如训练数据集、产出、日志等），通过环境变量的方式传递到容器中，您的脚本中可以读取这些变量。

| 变量名                 | 描述                                                         |
| ---------------------- | ------------------------------------------------------------ |
| `USER_CACHE_PATH`      | 用户缓存路径，配额20GB。训练、评测都会提供该变量，您可以使用来在训练、评测中共享文件。 |
| `TRAIN_DATA_PATH`      | 训练数据集路径                                               |
| `TRAIN_CKPT_PATH`      | 模型产出路径                                                 |
| `TRAIN_TF_EVENTS_PATH` | Tensorboard指标产出路径                                      |

**shell脚本中，可以按下面方式读取环境变量**

```shell
${TRAIN_DATA_PATH}
```

**python脚本中，可以按下面方式读取环境变量**

```python
import os
os.environ.get("TRAIN_DATA_PATH")
```

### 2. 产出规范

#### 2.1. 模型产出

a. 模型权重必须产出到 TRAIN_CKPT_PATH 指定的目录下；

b. 迭代的checkpoint文件放到一个以 global_step 开头的目录中，否则平台无法识别；

c. 目录名称只能包含字母(a-z, A-Z)、数字(0-9)、下划线(_)、中划线(-)、等号(=)、句号(.)等字符。

> 如果希望目录包含更多的参数信息，可以接着在 global_step 后面拼接，比如第20次迭代、lr=0.001、layer=2、head=1、hidden=50、maxlen=200的checkpoint，可以创建 `global_step20.lr=0.001.layer=2.head=1.hidden=50.maxlen=200` 的目录，并将权重保存到该目录下。

#### 2.2. Tensorboard指标

关于pytorch tensorboard的使用，可参考：

https://docs.pytorch.org/docs/2.7/tensorboard.html#

如下代码所示，参赛者可以读取环境变量`TRAIN_TF_EVENTS_PATH`，创建一个SummaryWriter对象

```python
from torch.utils.tensorboard import SummaryWriter
writer = SummaryWriter(os.environ.get('TRAIN_TF_EVENTS_PATH'))
```



## 模型评测官方要求

### 1. 环境变量

平台提供评测过程中，需要用到的目录（比如评测数据集、产出等），通过环境变量的方式传递到容器中，参赛者的脚本中可以读取这些变量。

| 变量名              | 描述                                                         |
| ------------------- | ------------------------------------------------------------ |
| `USER_CACHE_PATH`   | 用户缓存路径，配额20GB。训练、评测都会提供该变量，参赛者可以使用来在训练、评测中共享文件。 |
| `MODEL_OUTPUT_PATH` | 模型产出路径                                                 |
| `EVAL_DATA_PATH`    | 数据集、推理候选集、检索索引路径                             |
| `EVAL_RESULT_PATH`  | 产出物路径，存放中间结果和`result.json`                      |
| `EVAL_INFER_PATH`   | 选手上传推理脚本文件目录                                     |

**shell脚本中，可以按下面方式读取环境变量**

```Shell
${EVAL_DATA_PATH}
```

**python脚本中，可以按下面方式读取环境变量**

```Python
import os
os.environ.get("EVAL_DATA_PATH")
```

### 2. 产出规范

可以参考平台提供的脚本`infer.py`并实现推理函数，最终参赛者须在评测中返回`top10s`, `user_list`。

| 返回值      | 数据格式     | 说明                                                         |
| ----------- | ------------ | ------------------------------------------------------------ |
| `top10s`    | `list[list]` | 每一个参赛者推理出由10个`item_id`组成的list，所有参赛者的list组合成一个新的list<br>示例: `[[a1,a2...a10],[b1,b2...b10],...[n1,n2...n10]]` |
| `user_list` | `list[str]`  | `user_id`组成的list，其中`user_id`为str格式<br>示例: `['user_a','user_b',...'user_xxx']` |

> **注意**：`top10s`的第一个list应为`user_list`中第一个用户的序列推理出的结果，以此类推。若顺序不一致，会影响最终得分！！
